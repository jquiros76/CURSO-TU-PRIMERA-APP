¿Qué vas a aprender?

La calculadora de precios es un archivo HTML con 3 partes: el diseño (CSS), la estructura (HTML) y la lógica (JavaScript). Este mini curso te explica, zona por zona, qué puedes tocar sin romper nada.

📁
El archivo completo se llama calculadora-precios.html. Todo el código está en ese único archivo.

Textos visibles
Lo más fácil y seguro de cambiar

¿Dónde están los textos?

Todos los textos que el usuario ve están en el HTML, dentro de etiquetas como <h1>, <p>, <label>, etc. Puedes cambiarlos sin tocar CSS ni JavaScript.

html — textos que puedes cambiar 
<!-- Etiqueta verde pequeña arriba del título -->
<span class="badge">Herramienta gratuita</span>

<!-- Título principal -->
<h1>Calcula tu <span>precio</span> ideal</h1>

<!-- Descripción corta -->
<p class="subtitle">Ingresa el costo y tu margen...</p>

<!-- Etiquetas de los campos -->
<label for="costo">Costo del producto</label>
<label for="margen">Porcentaje de ganancia</label>

<!-- Texto del botón -->
<button>Calcular precio →</button>
💡
El texto entre las etiquetas de apertura y cierre (<h1>texto</h1>) es lo que aparece en pantalla. Cámbialo por el que quieras.

Símbolo de moneda

El símbolo $ aparece en 3 lugares. En el HTML (dentro del input) y en el JavaScript (en los resultados).

html + js — cambiar símbolo de moneda 
<!-- En el HTML (input del costo) -->
<span class="input-prefix">$</span>  <!-- cambia $ por ₡, €, Q... -->

// En el JavaScript (resultados)
document.getElementById('precio-venta').textContent = '$' + precioVenta;
document.getElementById('ganancia').textContent     = '$' + ganancia;
// Cambia '$' por '₡' en las dos líneas de arriba
✏️
Usa Ctrl+F en tu editor para buscar $ y encontrar todos los lugares rápido.

Colores y variables CSS
El "panel de control" del diseño

¿Qué son las variables CSS?

En lugar de escribir el mismo color 20 veces, lo defines una sola vez con un nombre, y lo reutilizas en todo el archivo. Si cambias el nombre, cambia en todos lados.

El bloque :root { } al inicio del CSS es donde están todas las variables.

css — variables de color (bloque :root) 
:root {
  /* Fondo y tarjeta */
  --bg:       #F0EDE8;   /* fondo beige de la página */
  --card:     #FFFFFF;   /* fondo blanco de la tarjeta */

  /* Colores principales */
  --accent:   #1A1A2E;   /* azul oscuro — botón y tarjeta resultado */
  --green:    #2D6A4F;   /* verde — texto de ganancia */
  --green-lt: #D8F3DC;   /* verde claro — fondo de ganancia */

  /* Secundarios */
  --muted:    #8A8A99;   /* gris — textos secundarios */
  --border:   #E4E0D8;   /* color de bordes */
  --error:    #C0392B;   /* rojo — mensajes de error */
}
🎨
Solo cambia los códigos hex #XXXXXX. Deja los nombres (--accent, --green…) exactamente igual — esos nombres los usa el resto del CSS.

La fórmula del cálculo
El cerebro de la calculadora — modificable con cuidado

¿Cómo funciona la fórmula?

Está en el JavaScript, dentro de la función calcular(). Son solo dos líneas matemáticas:

javascript — la fórmula 
// Paso 1: leer lo que el usuario escribió
const costo  = parseFloat(costoStr);   // convierte texto → número
const margen = parseFloat(margenStr);  // ej: "30" → 30

// Paso 2: calcular
const ganancia    = costo * (margen / 100);  // 100 * 0.30 = 30
const precioVenta = costo + ganancia;         // 100 + 30 = 130

¿Quieres agregar un costo de envío?

Ejemplo: suma un gasto fijo de $500 al precio de venta.

javascript — ejemplo con envío fijo 
const envio       = 500;                      // ← agrega esta línea
const ganancia    = costo * (margen / 100);
const precioVenta = costo + ganancia + envio;  // ← suma el envío
🔁
Regla de oro: No cambies los nombres de las variables (costo, margen, ganancia, precioVenta). Si los renombras, cámbialos en todos los lugares donde aparecen.

Zona de no tocar
Estas partes conectan todo — si las rompes, nada funciona

¿Por qué no tocar esto?

Estas líneas son los "cables" que conectan el HTML con el JavaScript. Si los cambias sin entenderlos, la calculadora dejará de responder al usuario.

Código¿Qué hace?¿Qué pasa si lo quito?
id="costo" Nombre único del input JS no encuentra el campo
id="margen" Nombre único del input JS no lee el porcentaje
id="resultados" Nombre de la sección resultado Resultado no aparece
onclick="calcular()" Conecta el botón con la función El botón no hace nada
display: none Oculta los resultados al inicio Resultados vacíos visibles
parseFloat() Convierte texto a número Sumas incorrectas (ej: 1+1=11)
html — partes críticas (no modificar) 
<!-- ❌ NO cambies los id= de los inputs -->
<input type="number" id="costo" .../>   <!-- id fijo -->
<input type="number" id="margen" .../>  <!-- id fijo -->

<!-- ❌ NO quites el onclick del botón -->
<button onclick="calcular()">Calcular</button>

<!-- ❌ NO toques el id ni el display:none de resultados -->
<div class="results" id="resultados">...</div>
⚠️
Regla: Si cambias un id= en el HTML, debes cambiar exactamente el mismo nombre en el JavaScript donde dice getElementById('ese-nombre').
❓ Cambias id="costo" por id="precio" en el HTML pero no cambias nada en el JavaScript. ¿Qué pasa?
